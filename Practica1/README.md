### Backup Completo
Se realizaron cinco respaldos completos por día con el objetivo de practicar y evaluar este tipo de estrategia. Este enfoque implica copiar toda la base de datos en cada respaldo, asegurando una copia íntegra en cada punto, pero con un mayor uso de almacenamiento y tiempo.

### Backup Incremental
El día 1 se efectuó un respaldo completo de la base de datos, seguido inmediatamente de un respaldo incremental. Durante los cuatro días siguientes, se realizaron únicamente respaldos incrementales, los cuales almacenan solo los cambios desde el respaldo anterior. Esta práctica permitió demostrar una alternativa más eficiente en términos de espacio y tiempo, aunque con un proceso de restauración más complejo.
